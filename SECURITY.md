## Reporting a Vulnerability

We take the security of Team Sync seriously. If you discover a vulnerability, we would like to know about it so we can take steps to address it as quickly as possible.

### How to Report

Please do **not** report security vulnerabilities through public GitHub issues.

Instead, please report them by sending an email to our security team or project maintainer at **security@example.com** (please replace this with your actual security contact email).

### What to Include

Please provide the following details in your report:
- A description of the vulnerability and its potential impact.
- The version(s) of Team Sync affected.
- Detailed steps to reproduce the vulnerability.
- Any proof-of-concept (PoC) code or screenshots, if applicable.

### What to Expect

1. You can expect an initial acknowledgment of your report within 48 hours.
2. We will investigate the issue and provide an update on whether the vulnerability is confirmed and accepted.
3. If confirmed, we will work on a fix and notify you when a patch or mitigation is ready.
4. Once the issue is resolved and a fix is released, you will be credited in our changelog/release notes (unless you prefer to remain anonymous).

Thank you for helping keep our project safe!
